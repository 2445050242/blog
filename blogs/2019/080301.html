  • # 数据的管理和传递state、props

    • 数据的变动最好放在最上层组件，然后通过数据props数据传递到下流组件

    • 不要直接修改state

      // 错误
this.state.comment = 'Hello';
/ 正确
this.setState({comment: 'Hello'});

    • 因为 this.props 和 this.state 可能是异步更新的，你不能依赖他们的值计算下一个state(状态),要弥补这个问题，使用另一种 setState() 的形式，它接受一个函数而不是一个对象。这个函数将接收前一个状态作为第一个参数，应用更新时的 props 作为第二个参数

      // 错误
this.setState({
  counter: this.state.counter + this.props.increment,
});
// 正确
this.setState((prevState, props) => ({
  counter: prevState.counter + props.increment
}));

  • # 事件处理

    • 在 React 中你不能通过返回 false 来阻止默认行为。必须明确调用 preventDefault

    • 一般情况下，如果你引用一个后面没跟 () 的方法，例如 onClick={this.handleClick} ，那你就应该 绑定(bind) 该方法。

      constructor(props) {
    super(props);
    this.state = {isToggleOn: true};
    
    // 这个绑定是必要的，使`this`在回调中起作用
    this.handleClick = this.handleClick.bind(this);
}

  • # 生命周期

    • componentWillMount：挂载之前
    • componentDidMount：挂载之后
    • componentWillUnmount 卸载
    • shouldComponentUpdate(nextProps, nextState)：你可以通过这个方法控制组件是否重新渲染。如果返回 false 组件就不会重新渲染。这个生命周期在 React.js 性能优化上非常有用。
    • componentWillReceiveProps(nextProps)：组件从父组件接收到新的 props 之前调用。
    • componentWillUpdate()：组件开始重新渲染之前调用。
    • componentDidUpdate()：组件重新渲染并且把更改变更到真实的 DOM 以后调用。

  • # 高级

    • 用户定义组件必须以大写字母开头，React 误认为小写是一个 HTML 标签，因为它首字母不是大写的

    • 使用 PropTypes 进行类型检查

      import PropTypes from 'prop-types';

class Greeting extends React.Component {
  render() {
    return (
      <h1>Hello, {this.props.name}</h1>
    );
  }
}

Greeting.propTypes = {
  name: PropTypes.string

  • # 其他

    • 一个常用的技巧是单一职责原则，即一个组件理想情况下只处理一件事。

    • http://huziketang.com/books/react/

    • 组件的 render 方法必须要用一个外层元素把内容进行包裹

    • 因为 class 是 JavaScript 的关键字,所以 React.js 中定义了一种新的方式：className 来帮助我们给元素添加类名

    • 因为 for 也是 JavaScript 的关键字，所以在 JSX 用 htmlFor 替代

    • 这些 on* 的事件监听只能用在普通的 HTML 的标签上，而不能用在组件标签上。

    • 当你调用 setState 的时候，React.js 并不会马上修改 state。而是把这个对象放到一个更新队列里面，稍后才会从队列当中把新的状态提取出来合并到 state 当中，然后再触发组件更新。

    • React.js 提供了一个属性 dangerouslySetInnerHTML，可以让我们设置动态设置元素的 innerHTML

      render () {
    return (
      <div
        className='editor-wrapper'
        dangerouslySetInnerHTML={{__html: '<h1>React.js 小书</h1>'}} />
    )
}

# 8.15 学习文档

  • 引入新的生命周期componentDidCatch(error, info)用于错误边界

  • this.myRef = React.createRef();绑定ref来操作DOM，使用this.myRef.current获取当前元素
  • {this.props.render(this.state)}渲染属性
  • 使用dangerouslySetInnerHTML 并通过 __html 键传递一个对象直接从 React 设置 HTML
