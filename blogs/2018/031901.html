# bootstrap-vue结合vue-cli构造响应式布局

  • # 安装

    • # 安装vue-cli基础框架
      cnpm install vue-cli -g 
vue init webpack my-demo 
cd my-demo 
cnpm install
npm run dev
      1
      2
      3
      4
      5
    • # 可以根据自己的需要修改vue-cli当中的配置，可以参考vue-cli的webpack模板项目配置文件分析,这里不再赘述

  • # 修改配置文件

    • # 修改package.json文件配置

      在package.json文件的dependencies的配置项中添加

      "bootstrap": "^4.0.0-beta.2",
"bootstrap-vue": "^1.2.0",
"popper.js": "^1.12.9"
      1
      2
      3
    • # 再次执行命令行命令cnpm install安装新增的依赖
    • # 在入口文件中添加bootstrap-vue依赖

      打开src/main.js文件，在头部引入依赖

      import BootstrapVue from "bootstrap-vue"
import "bootstrap/dist/css/bootstrap.min.css"
import "bootstrap-vue/dist/bootstrap-vue.css"
      1
      2
      3

      并将bootstrap-vue挂载在Vue实例上

      Vue.use(BootstrapVue)
      1

  • # 后续事宜

    • # UI框架文档推荐

      • bootstrap4官方文档

      • bootstrap-vue官方文档

        后期的编写可以使用这两个文档中的组件来进行快速开发，如果对组件有不清楚的地方可以查看文档了解对应的配置

    • # 建议
      • 能使用组件的地方尽量使用组件，毕竟在响应式布局这一块会比较稳定
      • 在使用自己编写的样式时需要考虑好不同浏览器屏幕宽度的情况下的响应式布局，这里可以去了解一下上面推荐的bootstrap4官方文档中的栅格布局，有用过bootstrap的同学应该就会知道了
      • 在编写样式过程中单位尽量使用rem,不理解rem的同学可以通过学习rem怎么计算来了解一下；当然这里可以自己直接根据根元素的字体大小来进行换算，也可以通过sass来自动计算
          @function px($size){
       @return ($size/40) + rem
    }
